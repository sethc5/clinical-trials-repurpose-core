## Last-Mile Module 1: Orphan Drug Designation (ODD) Petition

### Why this is the highest-priority build

The FDA's Orphan Drug Designation grants:
- **7-year market exclusivity** from approval date
- **$50M in tax credits** for Phase I and II trial costs
- **Waived FDA filing fees** (saves ~$3.5M at NDA stage)
- **Priority review** eligibility

Eligibility: any drug targeting a disease affecting fewer than 200,000 US patients. The entire pipeline is oriented around rare/niche diseases (ALS is the first instantiation, Chagas is planned — both qualify).

ODD applications are **10–20 pages** with a fixed FDA-defined structure. The FDA's guidance (21 CFR Part 316) defines exactly what's required in each section. This is not a creative document — it's a structured evidence assembly with regulatory-style narrative.

### What the ODD application requires

| ODD Section | Required content | Already in this repo |
|---|---|---|
| Drug description | INN name, structure, prior use | `db_utils.py` DrugEntry table |
| Disease prevalence | US patient count < 200,000 | Orphanet adapter + DisGeNET data |
| Medical plausibility | Why the drug would work | `evidence_synthesizer.py` output |
| Prior clinical experience | Existing safety/efficacy data | `evidence_extractor.py` + FAERS data |
| Bibliography | PubMed-cited evidence | `evidence_extractor.py` citation list |

**Every section maps directly to data already generated by the pipeline.** The gap is the renderer — a Jinja2 template that assembles this data into the FDA-specified section structure, plus an LLM call to convert the structured data into regulatory-style prose for the narrative sections.

### Build plan

```
last_mile/
    __init__.py
    odd_petition.py        # Core builder — ~200 lines
    templates/
        odd_petition.md.j2 # Jinja2 template with FDA section headers
    prompts/
        plausibility.txt   # LLM prompt: synthesizes mechanism evidence into narrative
        prevalence.txt     # LLM prompt: converts epidemiology data into FDA-style prose
```

**`odd_petition.py` logic:**

```python
class ODDPetitionBuilder:
    def __init__(self, drug_id: str, disease_id: str, db: RepurposeDB, llm_client):
        ...

    def build(self) -> str:
        # 1. Pull drug profile from db
        # 2. Pull disease prevalence from orphanet_adapter output in db
        # 3. Pull evidence synthesis from db (already generated by pipeline)
        # 4. Pull bibliography from evidence_extractor citations
        # 5. LLM call: render plausibility narrative from evidence_synthesis
        # 6. LLM call: render prevalence statement from epidemiology data
        # 7. Jinja2: assemble all sections into ODD template
        # 8. Return Markdown string (optionally render to PDF via weasyprint)
```

**Build effort:** ~2 days including the Jinja2 template, two LLM prompts, and integration test with a real drug-disease pair (e.g., riluzole × Chagas disease).

## Last-Mile Module 2: Clinical Trial Protocol (Phase II)

### Status: mostly built already

`compute/trial_designer.py` already generates a Phase II trial design narrative using Claude. It includes:
- Primary and secondary endpoints
- Patient eligibility criteria (incorporating biomarkers from `biomarker_identifier.py`)
- Dose escalation rationale (from `dose_analyzer.py`)
- Statistical power considerations

**What's missing to make it submission-ready:**

The current output is a narrative block. A clinical trial protocol submitted to FDA or an IRB follows a specific schema: CONSORT checklist sections, specific statistical analysis plan format, structured eligibility criteria table. The gap is format conversion — taking `trial_designer.py`'s narrative output and re-rendering it into the structured protocol template.

## OpenRouter swap (optional, recommended before productizing)

Currently every LLM call hardcodes `anthropic.Anthropic()`. OpenRouter allows model routing — cheaper models for boilerplate rendering, more capable models for reasoning-heavy synthesis. The change is minimal:

```python
# Current (in every compute module):
import anthropic
client = anthropic.Anthropic()

# OpenRouter swap:
from openai import OpenAI
client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key=os.environ["OPENROUTER_API_KEY"],
)
# model string changes from "claude-opus-4-20250514" to "anthropic/claude-opus-4"
```

Recommended routing:
- Narrative generation (plausibility, prevalence sections): `anthropic/claude-opus-4` (~$15/MTok)
- Boilerplate template rendering and formatting: `anthropic/claude-haiku-3-5` (~$0.80/MTok)
- Evidence extraction (high volume, many papers): `anthropic/claude-sonnet-4` (~$3/MTok)

## FTO handoff to patent-fto-core

When a repurposing candidate exits the pipeline, `clinical-trials-repurpose-core` can call the `patent-fto-core` service (separate repo) to check whether the repurposed use is claimed in existing patents. The API call is:

```python
import requests

fto_result = requests.post("http://patent-fto-core/fto/compound", json={
    "smiles": drug.canonical_smiles,
    "indication": disease.name,
    "use_type": "repurposed_indication",  # not synthesis — just method-of-use claims
}).json()
```

Method-of-use patents (claiming a known drug for a new indication) are the most common blocking risk in repurposing. `patent-fto-core` specifically handles method-of-use claim parsing, which differs structurally from composition-of-matter claims.
